Understanding Routers and Network Security

What is a Router?

A router is a device that forwards traffic between different IP subnets. It is commonly found in homes as the device connecting users to the internet. In business environments, organizations often deploy multiple routers to manage traffic within their infrastructure.

Routers and OSI Layer 3

Routers are considered Layer 3 devices, meaning they make routing decisions at the network layer of the OSI model. When routing functionality is integrated into a switch, these devices are often referred to as Layer 3 switches.

Routers and Network Types

Routers not only connect different IP subnets but also different network types. For example, they may link an internal Ethernet network to an external wide area network (WAN). Routers can also route traffic between different physical mediums, such as copper and fiber networks.

Security Features in Routers

Although routers are not inherently security devices, they include features that enhance security, such as Access Control Lists (ACLs). ACLs function similarly to packet filters in operating systems, allowing or denying traffic based on defined rules.

Access Control Lists (ACLs)

ACLs can be configured on a router's ingress or egress interfaces to filter incoming and outgoing traffic. These lists evaluate criteria such as source and destination IP addresses, port numbers, and protocol types, functioning similarly to firewall rules.

Example Network Topology

A typical network consists of users connected to a switch, which forwards traffic to a router. This router may connect to a WAN and route traffic to other subnetworks. ACLs can be applied at multiple points in this architecture to enhance security.

Preventing IP Spoofing with ACLs

One security concern is IP spoofing, where attackers manipulate packet headers to impersonate legitimate users. Routers can filter out traffic from reserved IP address ranges (e.g., RFC 1918 internal addresses) using ACLs, preventing unauthorized routing.

Reverse Path Forwarding (RPF)

RPF is a router security mechanism that ensures inbound traffic is responded to through the same interface it arrived on. This helps prevent IP spoofing attacks and mitigates man-in-the-middle and denial-of-service attacks.

Switches and OSI Layer 2

Switches primarily operate at OSI Layer 2, forwarding traffic based on MAC addresses. They use an application-specific integrated circuit (ASIC) to bridge traffic efficiently at high speeds.

Network Security Inside an Organization

Organizations typically deploy firewalls and intrusion prevention systems to secure external connections, but internal networks often lack these defenses. Threats such as unauthorized physical access to open switch ports or wireless connections remain concerns.

Port-Based Network Access Control (NAC)

To secure switch interfaces, organizations implement IEEE 802.1X, a port-based NAC standard requiring users to authenticate before gaining access to the network.

Switch Port Security Best Practices

  • Disable unused switch ports to prevent unauthorized access.
  • Monitor duplicate MAC addresses to detect MAC spoofing attempts.

Spanning Tree Protocol (STP)

Switch loops occur when multiple redundant links cause data packets to circulate indefinitely. Spanning Tree Protocol (STP), defined in IEEE 802.1D, prevents these loops by automatically blocking redundant paths.

Example of STP in Action

STP dynamically adjusts to network changes. If a primary link fails, STP recalculates the topology and enables backup links to restore connectivity while preventing loops.

Denial-of-Service (DoS) Attacks on Switch Networks

Attackers can flood a switch with excessive MAC addresses, exhausting its table and causing it to fail open, sending all traffic to all ports. Organizations implement flood guards to mitigate these attacks.

Layer 3 Switches

Layer 3 switches combine traditional switching capabilities with routing functions, forwarding traffic based on both MAC addresses (Layer 2) and IP addresses (Layer 3).

By understanding router and switch security, organizations can better protect their networks from internal and external threats.
