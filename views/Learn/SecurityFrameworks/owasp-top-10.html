Top 10 Web Application Security Risks (2021)

The 2021 OWASP Top 10 highlights key security vulnerabilities in web applications, with updates including new categories, revised definitions, and risk consolidations.

Updated OWASP Top 10 Risks

  1. Broken Access Control – Now the top risk, with 94% of tested applications showing issues. This includes improper user access restrictions and privilege escalation flaws.
  2. Cryptographic Failures – Previously “Sensitive Data Exposure,” this focuses on weak encryption practices leading to data breaches and system compromises.
  3. Injection Attacks – Still prevalent, with SQL injection, cross-site scripting (XSS), and command injection threats widely detected.
  4. Insecure Design – A new category addressing security flaws at the design level, emphasizing threat modeling and secure design principles.
  5. Security Misconfiguration – Includes outdated settings, mismanaged permissions, and improper cloud configurations. XML External Entities (XXE) issues are now merged into this category.
  6. Vulnerable & Outdated Components – Previously “Using Components with Known Vulnerabilities,” emphasizing risks from unpatched third-party libraries and outdated dependencies.
  7. Identification & Authentication Failures – Formerly “Broken Authentication,” now covering weak identity verification and session management flaws.
  8. Software & Data Integrity Failures – New category focusing on unverified software updates, insecure CI/CD pipelines, and data integrity risks. Includes Insecure Deserialization from 2017.
  9. Security Logging & Monitoring Failures – Previously “Insufficient Logging & Monitoring,” now expanded to cover broader detection and incident response gaps.
  10. Server-Side Request Forgery (SSRF) – Newly added based on community feedback, addressing risks where attackers manipulate server-side requests to access internal resources.

Why These Risks Matter

The OWASP Top 10 helps organizations prioritize security efforts by identifying the most critical vulnerabilities. Addressing these risks reduces the likelihood of breaches, improves compliance, and enhances overall web application security.
