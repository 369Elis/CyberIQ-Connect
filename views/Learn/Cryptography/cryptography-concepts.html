Cryptography

Cryptography involves techniques for securing and verifying data by transforming it to maintain confidentiality, integrity, and authenticity.

IBM i cryptographic services play a crucial role in protecting sensitive information, ensuring data integrity, authenticating users, and preventing denial of message transmission.

Cryptographic Algorithms

A cryptographic algorithm is a structured mathematical process designed to modify data, ensuring its security during transmission and storage.

Cipher Algorithms

Cipher algorithms encode readable data (plaintext) into an unreadable format (ciphertext) and convert it back when needed. There are two primary types:

Symmetric Encryption

This encryption method uses a single key for both encryption and decryption, making it efficient but requiring secure key distribution. A widely used example is the Advanced Encryption Standard (AES).

Types of symmetric encryption:
  • Block Cipher: Encrypts data in fixed-size blocks, processing a specific number of bytes at a time.
  • Stream Cipher: Encrypts data one bit or byte at a time, using a continuously generated keystream combined with the plaintext via XOR operations.

Asymmetric Encryption

Unlike symmetric encryption, asymmetric encryption uses two mathematically linked keys—a public key (shared openly) and a private key (kept secret). RSA is a commonly used asymmetric algorithm.

Although public-key encryption is more secure, it is significantly slower than symmetric encryption. Typically, asymmetric encryption is used to secure symmetric keys or create digital signatures.

Since algorithms are publicly available, data security relies on safeguarding encryption keys.

One-Way Hash Functions

A cryptographic hash function generates a fixed-length output (digest) from an input string of any length. A good hash function has the following properties:

  • Collision Resistance: Even small modifications in data result in entirely different hash values.
  • Irreversibility: It is computationally infeasible to reconstruct the original data from the hash.

Key Distribution Mechanisms

For encrypted data to be deciphered elsewhere, secure key transmission is necessary. Several techniques facilitate this:

  • RSA Encryption: A public key encrypts a symmetric key before sending it, which is then decrypted using the corresponding private key.
  • Diffie-Hellman Key Exchange: Both parties exchange specific parameters to generate matching symmetric keys independently.

Random Number Generation

Random numbers are essential in encryption, password salting, and key generation. IBM i uses a FIPS-approved pseudorandom number generator (PRNG) for cryptographic operations.

In cryptographic processors, an electronic noise source supplies unpredictable values for randomness. A FIPS 140-1 certified PRNG ensures robust security.

Cryptographic Operations

  • Message Authentication Code (MAC): Ensures message integrity by generating a unique code using a secret key and cipher algorithm.
  • Encryption & Decryption: Converts plaintext into ciphertext and vice versa using encryption keys.
  • HMAC (Hashed MAC): Uses a hash function combined with a secret key for authentication.
  • Digital Signatures: Used for authentication and non-repudiation.

Data Security Layers

  • Field-Level Encryption: The application manages key control and encryption operations.
  • Session-Level Encryption: Encryption happens at the communication layer without application involvement.
  • Link-Level Encryption: Data is encrypted at the lowest network layer.

Key Management

Effective key management is critical for cryptographic security. This includes storage, encryption, distribution, and conversion.

Key Storage

  • Cryptographic Services Keystore
  • Cryptographic Coprocessors
  • Digital Certificate Manager (DCM) Stores
  • Java Cryptography Extension (JCE) Keystore

Key Encryption & Conversion

To prevent unauthorized access, cryptographic keys are stored and transmitted in an encrypted format.

Financial PIN Security

Financial institutions use cryptography to generate and validate Personal Identification Numbers (PINs). PINs are securely assigned and processed using encryption techniques to verify user identity at ATMs and other financial platforms.
